# MT5Services 系统架构文档

## 版本历史
- **v1.0** (2026-03-10): 初始架构 - 基础数据采集和指标计算
- **v2.0** (2026-03-10): 中期优化 - 添加可靠性、监控和性能优化

## 系统概述

MT5Services 是一个个人单服务器量化交易系统，提供：
1. **数据采集**: 从MetaTrader 5获取实时市场数据
2. **指标计算**: 基于配置的指标计算引擎
3. **API服务**: RESTful API提供数据访问
4. **监控系统**: 系统健康状态监控和告警

## 核心组件

### 1. 数据采集层 (Ingestion Layer)
**位置**: `src/ingestion/`
- **BackgroundIngestor**: 后台数据采集器
- **数据源**: MT5终端实时数据
- **输出**: 写入内存缓存(MarketDataService)

### 2. 内存缓存层 (MarketDataService)
**位置**: `src/core/market_service.py`
- **功能**: 内存中缓存市场数据
- **数据类型**: Quote, Tick, OHLC
- **特点**: 只读缓存，由采集器写入

### 3. 指标计算层 (Indicators)
**位置**: `src/indicators/`
- **基础组件**:
  - `worker.py`: 原始指标计算工作器
  - `mean.py`, `momentum.py`, `volatility.py`, `volume.py`: 指标函数库
- **优化组件**:
  - `optimized/worker_enhanced.py`: 增强指标计算器（带缓存一致性检查）
  - `engine/pipeline_engine.py`: 指标流水线引擎（支持并行计算）
  - `engine/integration.py`: 指标集成器（向后兼容）

### 4. 配置管理系统 (Configuration)
**位置**: `src/config/`
- **基础配置**:
  - `compat.py`: 向后兼容配置接口
  - `centralized.py`: 集中式配置管理
- **高级配置**:
  - `advanced_manager.py`: 高级配置管理器（热加载、验证）

### 5. 事件存储系统 (Event Store)
**位置**: `src/utils/event_store.py`
- **功能**: SQLite持久化事件存储
- **优势**: 替代内存队列，事件重启不丢失
- **特性**: 自动重试、事件统计

### 6. 监控系统 (Monitoring)
**位置**: `src/monitoring/`
- **HealthMonitor** (`health_check.py`): 系统健康监控
- **功能**: 数据延迟、指标新鲜度、队列深度监控
- **告警**: 可配置阈值告警

### 7. 内存管理系统 (Memory Management)
**位置**: `src/utils/memory_manager.py`
- **功能**: 智能内存监控和清理
- **特性**: 缓存命中率统计、内存泄漏检测
- **策略**: 基于使用频率的智能清理

### 8. API层 (API Layer)
**位置**: `src/api/`
- **基础API**:
  - `market.py`: 市场数据API
  - `account.py`: 账户API
  - `trade.py`: 交易API
- **监控API**:
  - `monitoring.py`: 系统监控API
- **应用入口**:
  - `app.py`: 原始应用入口
  - `__init__.py`: 增强版应用入口

### 9. 存储层 (Storage Layer)
**位置**: `src/persistence/`
- **TimescaleWriter**: TimescaleDB写入器
- **StorageWriter**: 通用存储写入器

## 数据流

### 实时数据流
```
MT5终端 → BackgroundIngestor → MarketDataService → IndicatorWorker → TimescaleDB
      ↓                          ↓                      ↓
  实时数据                    内存缓存                指标计算
```

### 优化后数据流（v2.0）
```
MT5终端 → BackgroundIngestor → MarketDataService → IndicatorPipelineEngine → TimescaleDB
      ↓                          ↓                      ↓                      ↓
  实时数据                    内存缓存                并行指标计算          持久化存储
      ↓                          ↓                      ↓                      ↓
  事件存储 ←───────────────────┘                     监控系统 ←─────────────┘
   (SQLite)                                           (健康检查)
```

## 新增优化模块说明

### 1. 高级配置管理器 (`src/config/advanced_manager.py`)
**解决的问题**: 配置热加载、类型安全、环境变量覆盖
**关键特性**:
- 5秒轮询配置变更检测
- 自动类型转换 (int/float/bool/list/str)
- 环境变量优先级覆盖
- 配置schema验证

### 2. 智能内存管理器 (`src/utils/memory_manager.py`)
**解决的问题**: 内存泄漏、缓存管理、资源监控
**关键特性**:
- 实时内存使用监控
- 基于命中率的智能缓存清理
- 内存泄漏趋势检测
- 详细资源使用报告

### 3. 指标流水线引擎 (`src/indicators/engine/pipeline_engine.py`)
**解决的问题**: 指标计算性能、依赖管理、并行计算
**关键特性**:
- 指标依赖关系管理
- 多线程并行计算
- 计算图可视化 (Mermaid/Graphviz)
- 智能结果缓存

### 4. 事件存储系统 (`src/utils/event_store.py`)
**解决的问题**: 事件丢失、可靠性、重启恢复
**关键特性**:
- SQLite持久化存储
- 3次自动重试机制
- 完整事件统计
- 失败事件管理

### 5. 健康监控系统 (`src/monitoring/health_check.py`)
**解决的问题**: 系统监控、告警、故障检测
**关键特性**:
- 实时健康状态检查
- 可配置告警阈值
- 告警历史记录
- 手动解决功能

## 配置架构

### 配置文件结构
```
config/
├── app.ini              # 主配置文件（单一信号源）
├── market.ini           # 市场配置（继承app.ini）
├── ingest.ini           # 采集配置（继承app.ini）
├── indicators.ini       # 指标配置
├── db.ini              # 数据库配置
└── cache.ini           # 缓存配置
```

### 配置继承机制
```ini
# market.ini
[base]
extends = app.ini

[market]
# 覆盖或扩展app.ini中的配置
```

## 性能优化

### 计算性能优化
1. **并行计算**: 指标流水线支持多线程并行
2. **智能缓存**: 基于K线时间的缓存策略
3. **增量计算**: 只计算新K线的指标
4. **懒加载**: 按需加载配置和资源

### 内存优化
1. **智能清理**: 基于使用频率的缓存清理
2. **内存监控**: 实时内存使用监控
3. **泄漏检测**: 内存泄漏趋势分析
4. **资源回收**: 自动垃圾回收触发

### 可靠性优化
1. **事件持久化**: SQLite存储确保事件不丢失
2. **自动重试**: 关键操作支持自动重试
3. **健康检查**: 实时系统状态监控
4. **故障恢复**: 自动检测和恢复机制

## 监控和告警

### 监控指标
1. **数据延迟**: MT5数据到系统的延迟
2. **指标新鲜度**: 指标计算的时间延迟
3. **队列深度**: 待处理事件数量
4. **内存使用**: 进程内存使用情况
5. **缓存命中率**: 指标缓存效率
6. **计算成功率**: 指标计算成功比例

### 告警阈值
```yaml
数据延迟:
  警告: > 5秒
  严重: > 10秒
  
指标新鲜度:
  警告: > 30秒
  严重: > 60秒
  
内存使用:
  警告: > 80%
  严重: > 90%
```

## 故障排除指南

### 常见问题
1. **指标计算失败**: 检查pydantic依赖和配置文件
2. **内存泄漏**: 启用智能内存管理器监控
3. **数据延迟**: 检查MT5连接和网络
4. **配置不生效**: 检查配置热加载功能

### 诊断工具
1. **健康检查API**: `GET /monitoring/health`
2. **性能监控API**: `GET /monitoring/performance`
3. **内存报告API**: `GET /monitoring/memory/report`
4. **指标统计API**: `GET /monitoring/indicators/stats`
